## Description

The proxy has no command-line interface for specifying the downstream server and configuring its behavior.

## Requirements

### Downstream server specification

The downstream server is specified as positional arguments. The first argument without a leading `-` ends option processing; it and everything after it is the downstream spec. Auto-detection determines whether it is a URL or a command:

- If exactly one positional argument is provided and it starts with `http:` or `https:`, it is treated as an HTTP MCP server URL (SSE or streamable HTTP).
- Otherwise, all positional arguments are treated as the command and arguments to launch a local stdio MCP server subprocess.

At least one positional argument must be provided.

### Full flag reference

```
usage: immortal-mcp [options] <command> [args...]
       immortal-mcp [options] <url>

Reconnect:
  --reconnect-immediately
                          Reconnect to downstream immediately on disconnect (default: on-demand)
  --backoff-max SECS      Maximum backoff delay in seconds (default: 60.0)

Idle:
  --idle-timeout SECS     Disconnect downstream after N seconds of inactivity; 0 disables (default: 0)
  --idle-client-only      Only client→downstream messages count as activity
```

### Validation

- At least one positional argument (downstream spec) must be provided
- `--idle-timeout` and `--reconnect-immediately` are mutually exclusive; providing both is an error
- `--backoff-max` is only meaningful with `--reconnect-immediately`; providing it without is a warning (not an error)
- `--idle-client-only` without `--idle-timeout` is a warning (not an error)
- Numeric values must be positive
